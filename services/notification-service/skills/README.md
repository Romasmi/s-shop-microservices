# Developer & AI Skills

This directory contains machine-readable (and human-readable) instructions for developing in this project. 
It defines the patterns, rules, and workflows that ensure consistency across the codebase.

## How to use these skills

If you are an AI assistant:
- Read these files to understand how to implement new features.
- Follow the patterns defined here strictly.

If you are a Developer:
- Use these as a reference for adding new components.
- Keep these files up to date when patterns evolve.

## Skill Index

1. [Architecture](./architecture.md) - Project structure and Clean Architecture principles.
2. [API](./api.md) - Protobuf, gRPC Gateway, and code generation.
3. [UseCase](./usecase.md) - The Generic UseCase pattern and registration.
4. [Interface](./interface.md) - Implementing gRPC, CLI, and Kafka adapters.
5. [Infrastructure](./infrastructure.md) - Database and external service implementations.
6. [Domain](./domain.md) - Entities and repository interfaces.
